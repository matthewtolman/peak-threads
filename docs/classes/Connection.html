    Class Connection

    Represents a connection a shared thread has to another thread.

    Almost everything that is done with global methods in a Thread context are instead done on this object.

    These connection objects are unique per connection, and allow you to store connection data directly on them. Think of these as isolated channels.

    All shared worker handlers have these passed as the first parameter so that the handler can know who it's talking to and store any context information there.
